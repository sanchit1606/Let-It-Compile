# Results artifacts (tables)

This folder holds the reproducible CSV artifacts produced by each phase.

Guiding principle
- Each phase should produce at least one machine-readable artifact (usually a CSV) that is:
  - deterministic given the same code + seed + suite
  - accompanied by a clear schema (column meanings)
  - suitable for plots/tables in a paper

## Phase 0

- Output: `phase0_baseline.csv`
- Meaning: sweep over (kernel, size label, block size, reg cap) producing runtime + theoretical occupancy-related columns.

## Phase 1

- Output: `phase1_result.csv`
- Meaning: Nsight Compute (`ncu`) metrics collected for real kernel runs (may require Admin on Windows).

## Phase 3

Phase 3 is an RL environment, so the primary artifact is **rollout logs**.

- Step-level output: `phase3_rollout.csv`
  - One row per environment step (one attempted configuration)
  - Key columns:
    - `episode_id`, `episode_seed`, `step`
    - `kernel`, `matrix_size`, `block_size`, `reg_cap`
    - `time_ms`, `baseline_ms`, `speedup`, `reward`
    - `cupti_ok`, `cupti_reason`
    - `cupti_{metric}_norm` columns (normalized Phase 1 metrics, or 0 if unavailable)
    - `nvml_*` columns (normalized telemetry, or 0 if unavailable)

- Episode summary output: `phase3_episode_summary.csv`
  - One row per episode
  - Key columns:
    - `baseline_ms`, `mean_time_ms`, `mean_reward`, `best_speedup`

Generated by: `python experiments/phase3_rollout_log.py`
